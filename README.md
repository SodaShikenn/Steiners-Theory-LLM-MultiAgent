# Analysis of Interactions between LLM Multi-Agents

**LLM Multi-Agent Interaction Analysis using Steiner's Theory**

> Published at [NLP2025](https://www.anlp.jp/nlp2025/) (The 31st Annual Meeting of the Association for Natural Language Processing), March 2025

## Overview

This project analyzes interactions between multiple LLM agents from a social psychology perspective. We apply **Steiner's theory** --- a framework from social psychology that explains group problem-solving performance --- to LLM multi-agent (MA) systems.

We design quantitative evaluation metrics based on **Productivity Losses** and **Productivity Gains** from Steiner's theory, and analyze how inter-agent interactions affect overall group performance across different tasks and MA frameworks.

### Key Findings

- **Negative correlation between wrong idea selection and accuracy** across all conditions --- consistent with Steiner's theory that opinion diversity depends on the ability to select correct answers
- **Strong negative correlation between wrong idea generation and accuracy** --- propagation of incorrect information negatively impacts MA performance
- The effectiveness of interactions (correction, novel ideas) is **task-dependent and framework-dependent**

## Theoretical Background

**Steiner's Theory (1972):**

```
Actual Group Productivity = Potential Productivity
                          - Productivity Losses
                          + Productivity Gains
```

We defined the following quantifiable metrics for LLM multi-agent evaluation:

| Category | Metric | Description |
|---|---|---|
| **Productivity Losses** | Wrong Selection (誤選択) | Number of times agents select incorrect ideas from other agents |
| | Wrong Generation (誤生成) | Number of incorrect ideas generated by agents |
| | Same Opinion (同意見) | Number of times the same idea is repeated across agents |
| **Productivity Gains** | Correction (修正) | Number of times a wrong idea is corrected by another agent |
| | Novel Idea (新案) | Number of new ideas not present in single-agent responses |

## Experiment Setup

### Datasets
- **BBH (Causal Judgment)**: 200 questions --- a challenging reasoning task requiring causal inference
- **GSM8K (Mathematical Reasoning)**: 187 questions --- elementary-level math word problems

### Multi-Agent Frameworks
- **Cooperative** (Du et al., 2023): Two agents collaborate, sharing responses across rounds
- **Competitive** (Liang et al., 2024): Two agents debate from opposing viewpoints, with a judge deciding

### Model
- **GPT-4o** (OpenAI) as the backbone LLM for all agents

## Results

### Accuracy by Condition

| Condition | Accuracy (%) |
|---|---|
| GSM8K-SA-4o | 93.0 |
| GSM8K-Cooperative-4o | 97.5 |
| GSM8K-Competitive-4o | 94.5 |
| BBH-SA-4o | 69.5 |
| BBH-Cooperative-4o | 65.2 |
| BBH-Competitive-4o | 55.1 |

MA improves accuracy on easier tasks (GSM8K) but can hurt performance on harder tasks (BBH) --- highlighting the importance of analyzing inter-agent interactions.

## Getting Started

### Prerequisites
- Python 3.9+
- OpenAI API key (GPT-4o access)

### Installation

```bash
git clone https://github.com/SodaShikenn/LLM-Multi-Agent-Interaction-Analysis.git
cd LLM-Multi-Agent-Interaction-Analysis
pip install -r requirements.txt
```

### Configuration

```bash
cp .env.example .env
# Edit .env and add your OpenAI API key
```

### Running the Interactive Demo

```bash
cd src

# Cooperative framework
streamlit run cooperative_framework.py

# Competitive framework (debate)
streamlit run competitive_framework.py
```

### Running the Full Experiment Pipeline

```bash
cd src

# Step 1: Run single-agent baseline
python single_agent.py --dataset ../data/bbh.json --name bbh

# Step 2: Run multi-agent experiments
python batch_runner.py --dataset ../data/bbh.json --name bbh --framework cooperative
python batch_runner.py --dataset ../data/bbh.json --name bbh --framework competitive
python batch_runner.py --dataset ../data/gsm8k.json --name gsm8k --framework cooperative
python batch_runner.py --dataset ../data/gsm8k.json --name gsm8k --framework competitive

# Step 3: Evaluate metrics (Steiner's theory)
python evaluate_metrics.py \
    --ma_dir ../results/bbh_cooperative \
    --sa_dir ../results/bbh_single_agent \
    --dataset ../data/bbh.json \
    --output ../results/evaluation/bbh_cooperative_metrics.json

# Step 4: Analyze correlations (reproduce Tables 3-6)
python analyze_results.py --all_dir ../results/evaluation/
```
